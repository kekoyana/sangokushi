# ゲームについて
三国志を舞台としたSLGゲームです。

# 対象とするプラットフォーム
ブラウザーで動かす想定です。
PC・スマホ両方に対応させてください

# ゲームシステム
## マップについて
マップは以下に別れています。括弧は君主名です。
幽州(公孫瓚), 冀州(袁紹), 青州(孔融), 并州(劉備), 兗州(曹操), 司隸(董卓)、徐州(陶謙), 揚州(孫堅), 荊州(劉表), 交州(空白国), 益州(劉璋), 涼州(馬騰), 雍州(袁術)

## ゲームの開始
ゲーム開始で君主を選びます。

# 武将ステータス
- 武力(1-10): 戦争での強さ
- 知力(1-10): 政治や計略での強さ
- 兵士数: 兵力

# 州ステータス
- 金
- 兵糧
- 開発: 7月の兵糧の収入
- 商業: 1月の金の収入

# 政略フェーズ
## 画面構成
画面はマップエリア、情報エリア、コマンドエリア、メッセージエリアの4分割してください

## コマンド
### 政治
- 開発: 金を消費して州の開発度を上昇させる（兵糧収入増加）
- 商業: 金を消費して州の商業度を上昇させる（金収入増加）
- 徴兵: 金と兵糧を消費して兵士を雇用する
- 登用: 在野武将を配下に加える
- 外交: 他勢力との同盟・停戦交渉

### 軍事
- 攻撃: 隣接する敵州への攻撃
- 移動: 自軍武将・兵力の移動
- 守備: 州の守備を固める

### 人事
- 任命: 武将を州の太守に任命
- 配置: 武将の配置変更
- 褒賞: 武将の忠誠度を上げる

## ターン制
- 1ターン = 1ヶ月
- 各プレイヤーが順番にコマンドを実行
- ターン終了時に収入・支出の計算

## 収入システム
### 定期収入（ターン終了時）
- 金収入: 商業 × 10
- 兵糧収入: 開発 × 5
- 維持費: 兵士数 × 1（兵糧消費）

### 季節効果
- 春（3月）: 徴兵効率+20%
- 夏（6月）: 商業収入+10%
- 秋（9月）: 兵糧収入+20%
- 冬（12月）: 移動コスト+50%

## 戦闘システム
### 戦闘力計算
```
基本戦闘力 = （武将武力 + 知力/2） × 兵士数 / 100
```

### 戦闘結果
- 勝利: 敵州を占領、敵武将を捕虜または降伏
- 敗北: 兵力損失、武将負傷の可能性
- 引き分け: 両軍とも兵力損失

### 計略
知力の高い武将は以下の計略を使用可能：
- 火計: 敵兵力に大ダメージ
- 伏兵: 戦闘開始時に奇襲攻撃
- 混乱: 敵の戦闘力を一時的に低下
- 離間: 敵武将の忠誠度を下げる

## 勝利条件
全13州の統一

## 敗北条件
- 支配州が0になる
- 君主が死亡または捕虜になる
- 資金・兵糧が枯渇して維持不能

## イベントシステム
### ランダムイベント
- 豊作/凶作（兵糧収入変動）
- 疫病（兵力減少）
- 在野武将の登場
- 一揆・反乱

## UI仕様
### マップエリア
- 13州を色分け表示
- 各州に君主名、兵力、武将数を表示
- 選択州の詳細情報をハイライト

### 情報エリア
- 現在の年月表示
- プレイヤー勢力の総合情報
- 選択州の詳細ステータス

### コマンドエリア
- 実行可能コマンドをボタン表示
- コマンド実行時の確認ダイアログ
- 実行結果の表示

### メッセージエリア
- ゲーム進行に関するメッセージ
- 戦闘結果、イベント発生の通知
- 他勢力の動向

## 技術仕様
- フロントエンド: React + TypeScript
- 状態管理: Redux Toolkit
- スタイリング: CSS Modules
- ビルドツール: Vite
- レスポンシブデザイン対応（PC・モバイル）
